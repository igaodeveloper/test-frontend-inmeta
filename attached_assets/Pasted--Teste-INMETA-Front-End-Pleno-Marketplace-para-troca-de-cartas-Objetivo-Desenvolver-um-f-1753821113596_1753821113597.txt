# Teste INMETA – Front-End Pleno (Marketplace para troca de cartas)

## Objetivo
Desenvolver um front-end profissional, moderno, robusto, responsivo e elegante em React + TypeScript para um marketplace de troca de cartas, consumindo integralmente a API pública disponível em https://cards-marketplace-api-2fjj.onrender.com.

O foco é criar um SPA (Single Page Application) completo, modular, limpo, com alta qualidade técnica, UI/UX premium, animações fluidas, gerenciamento de estado eficiente, consumo seguro e perfeito da API, validação de formulários e tratamento robusto de erros.

---

## Tecnologias obrigatórias

- React 18+ com TypeScript
- Zustand para gerenciamento global de estado (incluindo autenticação e tema)
- React Query para cache, revalidação e consumo assíncrono da API
- React Router v6 para roteamento SPA com rotas públicas e privadas
- Tailwind CSS para estilização rápida, responsiva e customizada
- React Hook Form + Zod ou Yup para validação de formulários
- Axios para chamadas HTTP configuradas com interceptadores (incluindo JWT no header Authorization)
- Framer Motion para animações de transição e scroll
- Skeleton loaders e Toasts para feedbacks visuais e carregamento

---

## Funcionalidades exigidas (seguir 100% à risca)

1. **Cadastro de usuário**
   - Tela com formulário (nome, email, senha) com validação rigorosa
   - Envio para POST /register
   - Feedback visual claro para loading, sucesso e erro

2. **Login de usuário**
   - Tela com formulário (email, senha) e validação
   - POST /login para autenticar, armazenar token JWT em Zustand
   - Redirecionamento para dashboard após login
   - Feedback de erro e loading

3. **Dashboard do usuário logado**
   - Mostrar dados do usuário (nome)
   - Listar cartas do usuário (GET /me/cards)
   - Listar solicitações de troca criadas pelo usuário (GET /trades com filtro para as próprias)
   - Adicionar cartas à conta (POST /me/cards) com seleção da lista geral de cartas (GET /cards)
   - Criar solicitação de troca (POST /trades)
     - Selecionar cartas próprias para oferecer (type: OFFERING)
     - Selecionar cartas do sistema para receber (type: RECEIVING)
   - Deletar solicitações de troca criadas (DELETE /trades/:id)

4. **Marketplace público**
   - Página inicial pública que mostra lista paginada das solicitações de troca abertas (GET /trades)
   - Exibir cartas oferecidas e cartas desejadas para troca
   - Mostrar nome do usuário criador da troca
   - Navegação e filtros básicos (opcional, mas recomendado)

5. **Logout**
   - Limpar token e estado
   - Redirecionar para página pública

6. **Modo escuro (dark mode)**
   - Toggle de tema claro/escuro acessível em todas as telas
   - Armazenar preferência do usuário em Zustand + persistência local (localStorage)
   - Aplicar temas Tailwind via classes condicionais com transições suaves
   - Garantir acessibilidade e contraste adequado nos dois temas

---

## Requisitos técnicos detalhados

- SPA completa sem reloads, usando React Router
- Estado global para token JWT, info do usuário, autenticação e tema via Zustand
- Consumo API via React Query com cache, revalidação automática e tratamento de erros
- Axios configurado para enviar JWT no header Authorization em chamadas autenticadas, com interceptadores para refresh automático (se aplicável) e tratamento de erros globais
- Skeleton loaders nas listas e placeholders durante carregamento para melhor UX
- Tratamento completo de erros da API, exibindo mensagens claras e amigáveis ao usuário com toasts
- Design responsivo para mobile, tablet e desktop usando Tailwind CSS, com atenção para usabilidade em todos os dispositivos
- UI moderna, clean, com paleta harmoniosa e boa tipografia
- Animações suaves em scroll, entradas e transições de páginas com Framer Motion, incluindo animações screw em botões, cards e modais
- Formulários implementados com React Hook Form + validação por Zod/Yup, exibindo mensagens específicas de erro e desabilitando submit enquanto inválidos
- Componentização granular, reutilizável, com hooks customizados para lógica de negócio e integração com API
- Separação clara da camada de serviços de API (API clients, DTOs, handlers)
- Lazy loading e Suspense para otimização do carregamento inicial
- Barra de progresso global em navegação (exemplo: nprogress)
- Acessibilidade básica garantida (aria-labels, foco visível, navegação teclado, contrastes)

## Documentação da API para consumo

Base URL: https://cards-marketplace-api-2fjj.onrender.com

Endpoints principais:

- POST `/register` — Criar usuário
- POST `/login` — Autenticar e obter token JWT
- GET `/me` — Dados do usuário autenticado
- GET `/me/cards` — Cartas do usuário
- POST `/me/cards` — Adicionar cartas ao usuário
- GET `/cards` — Listar todas as cartas disponíveis
- GET `/cards/:id` — Detalhes da carta
- GET `/trades` — Listar solicitações de troca abertas
- POST `/trades` — Criar solicitação de troca
- DELETE `/trades/:id` — Deletar solicitação de troca
