# Especificação para projeto Front-End em React + TypeScript

## Requisito fundamental

**Todo o projeto deve ser 100% em português do Brasil (PT-BR).**

Isso inclui:

- Todos os textos visuais, labels, placeholders, mensagens de erro, avisos, botões, títulos, descrições e feedbacks para o usuário devem estar em português brasileiro claro, correto e formal.
- Qualquer documentação interna, comentários de código, mensagens em console e testes também devem estar em português.
- A linguagem do usuário final é o português do Brasil — nada de misturar inglês ou outras línguas.

## Requisito para valores monetários

- Caso o projeto apresente valores monetários, **eles devem estar sempre em Real Brasileiro (R$)**.
- Use o formato brasileiro padrão para moeda, por exemplo:
  - R$ 1.000,00
  - R$ 59,90
- Todos os cálculos, exibição e formatos numéricos relacionados a dinheiro devem seguir o padrão brasileiro (vírgula para decimais, ponto para milhares).
- Labels e descrições relacionadas a preços, valores, custos, pagamentos, saldos e outros devem estar adaptados para real brasileiro e texto em português.

## Objetivo

Garantir que toda a experiência do usuário e desenvolvimento do front-end estejam alinhados com o idioma e cultura brasileira, sem qualquer vestígio de outra língua.

## Contexto técnico

- Pode usar React, Vue, Angular ou outra stack, mas o idioma e formatação financeira devem respeitar a exigência acima.
- O foco é no front-end; o back-end e banco não fazem parte desta exigência, mas o front-end deve exibir e manipular tudo em português e Real.
- Validar e garantir que toda interação do usuário seja clara e natural para falantes do português brasileiro.

## Pontos extras

- Caso haja formulários, mensagens de validação, tooltips, modais, ou qualquer interface textual, tudo em português.
- Tratamento de moeda deve ser consistente em toda a aplicação.
- A usabilidade deve respeitar as convenções brasileiras, como formato de data, moeda e separadores numéricos.

---

## Exemplo rápido

- Botão: “Adicionar ao carrinho” (não “Add to cart”)
- Erro: “Email inválido” (não “Invalid email”)
- Valor: R$ 1.234,56 (não $1,234.56)

---

Com essas regras claras, o projeto será uma experiência totalmente brasileira e profissional, adequada para qualquer usuário do Brasil.
